## 2. Security Audit

### 2.1 ✅ Fixed in this audit

| ID | Severity | Issue | Fix |
|---|---|---|---|
| S-1 | P2 | `GET /api/files/[id]` — `id` URL param used in `startsWith`/`includes` with no validation. Empty `id` matches all files. | Strict hex regex `^[a-f0-9]{1,16}$` → 400 if invalid |
| S-2 | P2 | `POST /api/files/upload` — no max body size; multi-GB upload OOMs Node | Default 50 MB cap (`SCRIPT_MANAGER_MAX_UPLOAD_BYTES` override); 413 if exceeded |
| S-3 | P1 (real bug) | `PUT /api/scripts/[id]` — `body.name.length > 200` checked **after** `db.script.update()`. Oversized name partially commits. | Validate **before** update |
| S-4 | P1 (macOS) | `seed-local/route.ts` — hardcoded `LOCAL_SCRIPTS_DIR = '/home/z/my-project/local-scripts'` (Linux-only). macOS new-clone broken. | Env var `SCRIPT_MANAGER_LOCAL_DIR` → `<cwd>/local-scripts` → `~/script-manager/local-scripts` (resolved at request time) |
| S-5 | P3 (type safety) | `seed-local/route.ts` — `// @ts-nocheck` disabled type checking on 516 lines | Removed; all 30 SCRIPT_REGISTRY entries pass strict checking |

### 2.2 ⚠️ Remaining issues (not fixed; documented for follow-up)

| ID | Severity | Issue | Recommendation |
|---|---|---|---|
| S-6 | P2 | `POST /api/llm-config` — `apiKey` stored as **plaintext** in SQLite. db file leak = key compromise. | Encrypt with `crypto.createCipheriv` (AES-256-GCM); key from `SCRIPT_MANAGER_ENCRYPTION_KEY` env. List endpoint already strips `apiKey` from GET response — good. |
| S-7 | P3 | `POST /api/seed?source=github` — hardcoded GitHub script list (40 files) rather than dynamic. New scripts in upstream repo require manual update here. | Use `https://api.github.com/repos/{owner}/{repo}/contents` to list dynamically |
| S-8 | P3 | `/api/execute` — `python3 script.py` is sandboxed only by file system. Scripts have full read/write to `cwd` and env vars. | Add `isolate-vm` or `bwrap` wrapper for untrusted scripts. (Out of scope for personal-use deployment.) |
| S-9 | P3 | `POST /api/ai/analyze-script` — script content goes into LLM prompt; **no system prompt isolation**. A malicious script could inject instructions. | Wrap user content in a `user` role message; never mix into system prompt. Add length cap (currently unbounded). |
| S-10 | P3 | `next.config.ts` likely has `ignoreBuildErrors: true` (confirmed — TS errors didn't break build). Hides future regressions. | Set to `false` once lint pass reaches 0 errors. |
